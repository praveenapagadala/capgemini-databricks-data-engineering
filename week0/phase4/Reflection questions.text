1. Why is cleaning done before joining tables?
Cleaning is done before joining to ensure the data is accurate and consistent. If the data contains null or invalid values, the join results may become incorrect.

2. What would go wrong if null keys are not removed?
If null keys exist, joins may fail to match records correctly, which can lead to missing data or incorrect aggregation results.

3. How did you decide join order?
I joined smaller or filtered datasets first to reduce data size, and then joined larger tables. This improves performance and makes processing faster.

4. Which step was most difficult and why?
Handling missing values and incorrect data types was the most challenging because they affected transformations and aggregations.

5. How is SQL logic similar to PySpark?
SQL operations like SELECT, WHERE, JOIN, and GROUP BY directly map to PySpark functions such as select(), filter(), join(), and groupBy().

6. What challenges will appear with large data?
Large datasets can cause performance issues, longer processing time, and memory limitations, requiring distributed processing and optimization.

7. Can you explain your pipeline in simple steps?
The pipeline reads raw data, cleans and filters invalid records, joins related datasets, performs aggregations, and produces the final output for analysis.
