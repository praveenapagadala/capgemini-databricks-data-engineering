# Phase 1 – SQL to PySpark Basics
The goal of phase 1 was to understand with the basics of operations within PySpark, which was necessary for understanding how these operations could relate to SQL queries.

## Concepts Learnt
While executing this phase, I have gained some key concepts of working with PySpark dataframes. Specifically, in this section, I have practised working with the following topics:
* DataFrame (Table)
* Viewing data using "show()"
* Selecting columns (select())
* Filtering rows (filter())
* Grouping data (groupBy())


## SQL vs PySpark Mapping
Another significant point I had to consider when going through this phase is that SQL queries should be able to be mapped to PySpark commands.

## Dataset Used
This section used the **customer** dataset with the following columns:
* customer_id
* customer_name
* city
* age

A set of sample data was provided including customer named Ravi residing in Hyderabad and another one called Sita from Chennai.

## Tasks Accomplished
During this phase, the following tasks were accomplished:
1. Displaying all customer datasets
2. Filtering out data by city
3. Filtering data by age
4. Selection of some columns
5. Count of customer groups by city

All these tasks help to gain experience of basic operations within PySpark.

## Output Produced

The output of this phase includes:
* Full customer datasets displayed
* Customer data filtered by different criteria
* Selection of particular columns
* Aggregation operation to display the number of customers in each city

## Key Learning Points
* Operations with PySpark DataFrames resemble operations within SQL tables
* Operations within PySpark resemble SQL queries
* Basic PySpark operations include select, filter, and groupBy commands
